Parse foo.github.io to foo.com

Nov 19 2018

Description

In theory, static pages that don’t depend on the database, such as personal blogs, can be mounted on github pages.

Prepare

  • Personal domain name
  • Github pages such as foo.github.io

Step

  1. Create CHAME file in blog/source
  2. Type your personal domain such as foo.com in file CHAME

    don’t type www!

  3. Add a parsing record in the domain providers console, most domain providers need the following fields.
    1. Record type: CHAME
    2. Host Record: www | @, this means we need to add two different host records to parsing records.
    3. Parse line: default
    4. Record value: foo.github.io, that is your github pages.
    5. TTL: 10 min, prasing interval
  4. All the operations have been completed by this step, maybe you have to wait a few minutes because of some domain providers mechanisms problem without aliyun.
