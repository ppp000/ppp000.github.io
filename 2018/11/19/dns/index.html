foo.github.io解析到foo.com

Nov 19 2018

description

gihub pages仅支持静态页面，博客恰好也属于其中，理论上说，不依赖于数据库的静态页面就可以在没有服务器的情况上访问。

准备

  • 个人域名
  • 可正常访问的github个人博客，如：foo.github.io

步骤

  1. blog/source下新建CHAME文件
  2. 写上：foo.com 不要www不要www不要www
  3. 打开域名商的控制台，添加解析记录，大多域名商需要以下几个字段
    1. 记录类型：CNAME
    2. 主机记录: www | @, 也就是说我们需要添加两条解析记录！
    3. 解析线路：默认
    4. 记录值：foo.github.io，即是你的目标域名
    5. TTL: 10分钟，dns解析间隔
  4. 到这一步已经完成所有操作了，部分域名商可能需要等待几分钟，阿里云可以秒解析
